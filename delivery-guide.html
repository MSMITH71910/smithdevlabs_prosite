📋 Client Intake — Collect This Before You Build

🔢 Step-by-Step Build Instructions

1
Set up GitHub repository
Go to github.com → New Repository → name it clientname-website → set to Public → Initialize with README. Clone to your computer with: git clone [url]
2
Create folder structure
Inside the repo folder create: /css/style.css, /js/main.js, /images/ folder, then: index.html, services.html, about.html, contact.html, and one optional 5th page (Gallery, FAQ, or Testimonials)
3
Build the navbar and footer first
Create a consistent navbar with logo, nav links, and a CTA button ("Call Now" or "Book Free Estimate"). Build the footer with address, phone, email, and social links. Copy this to all pages — consistency is everything.
4
Build the Home page (index.html)
Hero section: big headline, subheadline, 2 CTA buttons ("Call Now" + "Get a Free Quote"). Services overview grid (3–4 cards). Why Choose Us section. Contact/CTA section at bottom.
5
Build Services page
List each service with a short description, icon or image, and a bullet list of what's included. End with a CTA: "Ready to get started? Call us today."
6
Build About page
Owner photo (or team photo), business story, years in business, licenses/certifications, service area map or list. Include trust signals: BBB badge, license number, insurance mention.
7
Build Contact page with Netlify Form
Add a form with: Name, Email, Phone, Service Needed, Message. Add this to the form tag: data-netlify="true" name="contact". This makes Netlify capture submissions automatically — zero backend needed.
8
Add Google Analytics
Go to analytics.google.com → Create property → Get the G-XXXXXXXX tracking ID → Paste the script tag in the <head> of every page. This lets client see how many people visit.
9
Optimize images
Go to squoosh.app → Compress every image to under 200KB. Large images slow the site and hurt SEO. Name images descriptively: smith-plumbing-philadelphia.jpg (not IMG_4832.jpg).
10
Push to GitHub and deploy on Netlify
Run: git add -A → git commit -m "Initial build" → git push. Then go to netlify.com → Add new site → Import from Git → Select repo → Deploy. Site goes live in 60 seconds at a random netlify URL.
11
Connect client's custom domain
In Netlify → Site Settings → Domain Management → Add custom domain → Enter domain name. Then go to wherever domain is registered (GoDaddy, Namecheap, Google) → DNS settings → Add Netlify's nameservers or CNAME record. Takes 5–30 min to propagate.
12
Test everything — use the testing checklist below
Check every page on mobile AND desktop. Submit the contact form and verify you receive the notification email. Check all links work. Run through Google PageSpeed Insights (pagespeed.web.dev) and fix any red issues.
13
Record a Loom walkthrough video (5–10 min)
Go to loom.com → Record screen → Walk through every page of the live site. Explain what each section does and how to contact you for changes. This is your professional delivery — clients LOVE it. Share the Loom link in your delivery email.
14
Send delivery email + collect final payment
Use the Project Delivery Email template (Section 15). Send final 50% Wave invoice at the same time. Do not hand over GitHub repo access until paid in full.

🔢 Step-by-Step: Website Build (Days 1–7)

1
Build the 10-page website (same as Starter steps 1–12)
Pages: Home, Services (main), Service Detail pages (2–3 individual service pages), About, Gallery/Portfolio, FAQ, Blog (optional), Contact, Thank You page (after form submit)
2
Create Chatbase account and new chatbot
Go to chatbase.co → Sign up free → Click "New Chatbot" → Name it after the client's business (e.g. "Rivera's Plumbing Assistant")
3
Upload business knowledge to Chatbase
In Chatbase → Sources tab → Add Text source → Paste in: all services + descriptions, full FAQ list, pricing info, hours, address, service area, booking instructions. Also add the website URL as a web crawl source.
4
Configure chatbot personality and greeting
Chatbase → Settings → Instructions: write a system prompt like: "You are [Business Name]'s friendly AI assistant. Answer questions about our services, pricing, and availability. Always be helpful and professional. If asked something you don't know, say: 'Great question! Let me connect you with our team — can I get your name and phone number?'"
5
Set up lead capture in Chatbase
Chatbase → Leads tab → Enable lead collection → Set fields: Name (required), Email (required), Phone (required). Set the prompt: "Before I connect you with our team, could I get your name, email, and phone number?"
6
Build Make.com automation: Chatbase lead → CRM + email alert
Go to make.com → Create new scenario → Add trigger: Webhooks → Custom webhook (copy the URL) → Paste URL in Chatbase → Integrations → Webhook. Add actions: 1) Gmail → Send email to owner with lead details. 2) Airtable → Create record with Name/Email/Phone/Date columns.
7
Get chatbot embed code and add to all website pages
Chatbase → Embed tab → Copy the script tag → Paste it just before </body> on every HTML page. The chat bubble will appear automatically on all pages.
8
Test the chatbot with 15+ real questions
Ask it: hours, pricing, services, emergency availability, how to book, location, parking, payment methods. Check every answer. Fix bad answers by adding better content to Sources. Submit a test lead and verify it appears in Airtable and triggers owner email.

🔢 Automation 1: Lead Form → Owner SMS + CRM Entry

1
Set up Twilio account

2

3
Test: Submit website contact form → verify SMS arrives in under 60 seconds
Use a real phone number for testing. Check Make.com execution history to see if the scenario ran. If it failed, check the error message in Make.com.

🔢 Automation 2: Job Complete → Google Review Request

1
Get client's Google Business Profile review link
Go to Google Maps → Search their business → Click "Write a review" → Copy the URL. Shorten it with bit.ly. This is what you send customers.
2
Create Make.com scenario triggered by Airtable status change
Trigger: Airtable → Watch Records → When "Status" field changes to "Job Complete". Actions: 1) Twilio → Send SMS to customer: "Hi {{name}}, thanks for choosing [Business]! If we did a great job, would you mind leaving us a quick Google review? It really helps: [link]". 2) Gmail → Send same message as email.
3
Train client how to mark jobs as "Complete" in Airtable
Record a 2-minute Loom video showing: open Airtable → find client row → change Status dropdown to "Job Complete" → automation fires. This is their one step to trigger the review request.

🔢 Automation 3: Appointment Confirmation + 24hr Reminder

1
Connect Calendly to Make.com
Make.com → New scenario → Trigger: Calendly → Watch Events (new booking). This fires every time someone books an appointment through Calendly.
2
Send immediate confirmation SMS + email
Action: Twilio SMS: "Hi {{invitee_name}}, your appointment with [Business] is confirmed for {{start_time}}. Address: [address]. Questions? Call [phone]. See you then!" Also send Gmail confirmation email.
3
Schedule 24-hour reminder
Add a Sleep module set to 23 hours (Make.com can pause scenarios). After 23 hours: Twilio SMS: "Reminder: Your appointment with [Business] is tomorrow at {{start_time}}. Address: [address]. Reply CANCEL to cancel or call [phone]."

🔢 Automation 4: 3-Email Lead Follow-Up Sequence

1
Set up Mailchimp audience and email templates
Mailchimp → Audience → Create audience "New Leads". Create 3 email templates: Day 1 (intro + offer), Day 3 (follow-up + FAQ), Day 7 (last chance + testimonial).
2
Build Make.com scenario to add leads to Mailchimp sequence
Trigger: Webhooks (same webhook as lead form). Action: Mailchimp → Add/Update Subscriber → Add to "New Leads" audience → Tag as "new-lead". The Mailchimp automation sequence fires from there.
3
Build Mailchimp Customer Journey automation
Mailchimp → Automations → Customer Journeys → Starting point: Tag added "new-lead" → Send Email 1 immediately → Wait 2 days → Send Email 2 → Wait 4 days → Send Email 3. Activate the journey.
